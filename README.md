# TSP-team project

Структура проекта 
---


```text
TSP-team/
├── tsp/
│   ├── __init__.py

│   ├── route_types.py      # описание точек маршрута
```

Постановка задачи
---
Требуется построить туристический маршрут по заданному множеству точек, учитывая:
- ограничение по времени
- ограничение по бюджету
- минимум 2 точки питания
- точки питания не должны идти подряд

Проблема
---
Нельзя просто использовать Lin–Kernighan, так как классический алгоритм Lin–Kernighan решает задачу коммивояжёра (TSP) следующим образом:

- требуется посетить все вершины
- найти гамильтонов цикл (обход всех вершин с возвратом)
- минимизировать длину маршрута

алгоритм предполагает, что множество вершин уже задано и фиксировано, то есть идти случайно (как мы хотели) не получится

также алгоритм не умеет "выкидывать" точки из маршрута

Решение
---

1. Выбор подмножества подходящих точекб таких,что:

- суммарное время ≤ лимита
- суммарная стоимость ≤ бюджета
- содержит ≥ 2 точки категории еды
  
Используем жадный алгоритм:
старт из начальной точки, далее добавляем точки, минимизируя: расстояние, время посещения, стоимость + приоритетно добавляем точки еды

2. На выбранном множестве точек строим начальный маршрут (nearest neighbor), применяя Lin–Kernighan:

3. Корректировка маршрута:

- LK дает цикл, поэтому убираем возврат в начало
- После оптимизации проверяем бюджет и время и "правило еды"
- Если нужно обрезаем маршрут или переставляем точки


Вывод:
---
Поскольку классический алгоритм Lin–Kernighan предназначен для задачи коммивояжёра, в которой требуется посетить все вершины, он не может напрямую применяться к задаче персонализации туристического маршрута с ограничениями по времени и бюджету. 

Поэтому мною был использован двухэтапный подход: на первом этапе формируется допустимое подмножество достопримечательностей (с учетом выполнения всех ограничений по цене и времени), а на втором этапе порядок их обхода оптимизируется с помощью алгоритма Lin–Kernighan.
